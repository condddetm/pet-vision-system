\chapter*{摘要}
\addcontentsline{toc}{chapter}{摘要}

\begin{abstractbox}
\noindent \textbf{背景与目标}\ 
本项目面向课程大作业\textbf{挑战项 D（多任务视觉分析）}，基于 Oxford-IIIT Pet
公开数据集（37 类宠物品种，7{,}390 张图像），构建了一个集成\textbf{图像分类}与
\textbf{图像分割}的端到端多任务系统，并通过 Web 可视化界面向用户提供完整的
数据探索、模型推理、实验分析与报告导出能力。

\vspace{0.3em}
\noindent \textbf{方法}\ 
分类任务采用 \textbf{ResNet-34}（ImageNet 预训练）作为主模型，并同时训练
\textbf{ResNet-34（随机初始化）}与\textbf{自定义 SimpleCNN（4 层 CNN）}两个基线用于对比；
分割任务采用 \textbf{U-Net} 架构，编码器基于同一 ResNet-34 主干。所有模型在
相同的 7:2:1 固定种子划分上训练与评估，确保结果可对比、可复现。此外，在
ResNet-34 最后一个残差块上注册 Hook 反向传播，生成 \textbf{Grad-CAM}
类激活热力图以辅助模型可解释性分析。

\vspace{0.3em}
\noindent \textbf{主要结果}\ 
ResNet-34 主模型在测试集上取得 \textbf{Accuracy = 0.9865}，Precision/Recall/F1
均为 \textbf{0.9865--0.9872}；U-Net 分割模型取得 \textbf{Dice = 0.9374}、
\textbf{mIoU = 0.8895}、\textbf{Pixel Accuracy = 0.9709}。三模型对比实验表明
迁移学习带来 \textbf{+30.3\%} 的分类准确率提升，并验证了 SimpleCNN
作为轻量基线的容量局限。

\vspace{0.3em}
\noindent \textbf{系统实现}\ 
系统采用前后端分离架构：后端为 FastAPI + PyTorch，提供 14 个 REST API 与 1 个
WebSocket 端点；前端为 Vue 3 + Element Plus + ECharts，包含 5 个功能页面。
实现了\textbf{三模式推理}（单张/批量/三模型对比）、\textbf{实时摄像头识别}、
\textbf{Grad-CAM 可视化}、\textbf{PDF 多页报告导出}、\textbf{WebSocket 流式推理日志}、
\textbf{错误反馈闭环}与\textbf{中英双语切换}等多项扩展能力。

\vspace{0.3em}
\noindent \textbf{关键词}：多任务学习、图像分类、语义分割、ResNet、U-Net、
迁移学习、Grad-CAM、Vue 3、FastAPI。
\end{abstractbox}

\section{课程任务与研究目标}

本项目对应课程大作业的\textbf{挑战项 D：多任务视觉分析}。任务要求在同一系统内
实现\textbf{图像分类}与\textbf{图像分割}两种计算机视觉任务，基于公开数据集训练、
评估并对比不同模型的性能，同时构建一个面向用户的可视化交互系统，覆盖从数据
加载、模型推理、指标展示到实验分析的完整工作流。

具体而言，本项目需要完成：

\begin{enumerate}[leftmargin=*]
  \item \textbf{双任务模型}\ — 分类器实现 37 类宠物品种识别；分割器输出宠物前景的
        像素级二值掩膜。二者共享数据加载与预处理管线，支持单次推理同时输出两类结果。
  \item \textbf{完整训练流程}\ — 包括数据划分、训练脚本、训练过程曲线记录、
        以及独立测试集上的定量评估与混淆矩阵分析。
  \item \textbf{对比实验}\ — 验证迁移学习相对从零训练的收益，并设立自定义轻量基线
        定位 ResNet 架构容量的实际贡献。
  \item \textbf{可视化系统}\ — 以 Web 页面向用户交互式呈现数据集、推理过程、
        训练指标与对比分析；支持图片上传、实时摄像头抓拍等多种输入方式。
  \item \textbf{加分扩展}\ — 包括可解释性分析（Grad-CAM）、Top-K 预测、
        报告导出、错误反馈闭环、实时日志流、国际化支持等。
\end{enumerate}

\section{研究意义}

宠物品种识别与前景分割虽是经典的计算机视觉子任务，但同时具备以下特点
使其适合作为多任务学习的教学载体：

\begin{itemize}[leftmargin=*]
  \item \textbf{细粒度挑战}\ — 37 类品种中部分品种外观差异微小（如缅因猫与
        挪威森林猫），考验模型的细粒度特征提取能力。
  \item \textbf{类别不均衡}\ — 各品种样本数在 184--200 之间，相对均衡，
        但仍存在少量类别样本偏少，便于观察不均衡对结果的影响。
  \item \textbf{标注完整}\ — 数据集提供像素级 trimap 分割标注，可直接用于分割训练，
        无需额外人工标注。
  \item \textbf{语义丰富}\ — 宠物的姿态、光照、遮挡、背景等变化丰富，
        天然适合验证数据增强与模型鲁棒性。
\end{itemize}

在本项目实施过程中，还需兼顾\textbf{工程化落地}：模型推理需要通过稳定的 API 对外
提供服务，前端需能实时反馈推理过程与结果，系统整体需具备可复现性与可部署性。
这些要求使项目从单纯的深度学习实验扩展为一个面向真实场景的小型智能视觉系统。

\section{主要工作与贡献}

本报告系统梳理项目实施过程中的关键技术决策与实验结果，主要工作包括：

\begin{enumerate}[leftmargin=*]
  \item 在 Oxford-IIIT Pet 数据集上训练 ResNet-34 分类模型，测试集 Accuracy 达
        \textbf{0.9865}；训练 U-Net 分割模型，测试集 Dice 达 \textbf{0.9374}。
  \item 对比三种分类方案（ResNet-34 预训练 / 随机初始化 / SimpleCNN）与两种
        分割方案（U-Net 编码器预训练 / 随机初始化），验证迁移学习的决定性收益。
  \item 实现 Grad-CAM 类激活图生成流程，可视化模型在 \texttt{layer4[-1]}
        的判别依据区域。
  \item 构建完整的前后端分离 Web 系统，提供 14 个 REST API + 1 个 WebSocket 端点，
        覆盖单张推理、批量并发、三模型对比、PDF 导出等核心交互。
  \item 实现一系列加分扩展能力：实时摄像头识别、错误反馈闭环、中英双语、
        多页 PDF 报告导出等。
  \item \textbf{工程化}\ — 完成工程化落地：Docker 容器化部署、API 限流、CORS 加固、
        前后端自动化测试（pytest + vitest，共 34 用例）与生产构建优化。
\end{enumerate}

Visual Geometry Group 发布。该数据集包含 37 类常见宠物品种（25 类犬科 + 12 类猫科），
总计约 7{,}390 张 JPEG 图像及对应的像素级 trimap 分割标注。

\subsection{数据组织}

原始数据按如下目录组织：

\begin{lstlisting}
data/pets/
|-- images/                      # 7390 张宠物 JPEG 图像
|       Abyssinian_1.jpg ...
|       samoyed_200.jpg
`-- annotations/
    |-- trimaps/                 # 像素级分割标注 (PNG)
    |       Abyssinian_1.png ...   # 1=前景, 2=背景, 3=边界
    |-- list.txt                 # 官方完整清单
    |-- trainval.txt             # 官方训练/验证清单
    `-- test.txt                 # 官方测试清单
\end{lstlisting}

其中 \textbf{trimap} 是三值掩膜（1 前景 / 2 背景 / 3 边界过渡区），
本项目将其映射为二值前景分割目标：前景 = 1，背景与边界 = 0。

\section{数据划分}

为保证分类与分割实验的可比性与可复现性，本项目采用\textbf{固定随机种子
（seed = 42）的 7:2:1 划分}策略。划分逻辑集中在 \texttt{training/split\_utils.py}
中实现，所有训练脚本调用相同接口以确保：

\begin{itemize}[leftmargin=*]
  \item 训练集 / 验证集 / 测试集在所有实验中完全相同；
  \item 三模型对比、迁移学习对比等均在同一测试集上评估；
  \item 新增脚本可无需额外处理即获得一致划分。
\end{itemize}

划分后各子集在类别分布上保持大致均衡（每类测试样本数约 20 张）。

\section{预处理}

\subsection{分类任务}

分类任务将所有输入统一缩放为 $224 \times 224$，采用 ImageNet 数据集的标准化参数：

\begin{equation}
  x' = \frac{x / 255 - \mu}{\sigma},\quad
  \mu = [0.485, 0.456, 0.406],\quad
  \sigma = [0.229, 0.224, 0.225]
\end{equation}

选择 $224 \times 224$ 尺寸的原因在于：（1）与 ResNet 的 ImageNet 预训练输入一致，
便于迁移学习；（2）兼顾显存占用与细粒度特征保留。

\subsection{分割任务}

分割任务采用 $256 \times 256$ 作为输入尺寸，略大于分类的 224，以保留更多的

\section{整体方案设计}

本项目属于典型的\textbf{多任务视觉分析}范式：将两个异构视觉任务（分类 + 分割）
通过合理的模型架构与共享主干组合在同一系统中，既保证各任务的性能，又降低总体
资源开销。具体选型如下：

\begin{itemize}[leftmargin=*]
  \item \textbf{分类任务}\ — 采用 ResNet-34（2015）\cite{he2016resnet} 作为主干，
        在 ImageNet 上预训练权重基础上进行 fine-tuning。
  \item \textbf{分割任务}\ — 采用 U-Net（2015）\cite{ronneberger2015unet} 架构，
        编码器替换为与分类任务一致的 ResNet-34，实现主干共享的思路。
  \item \textbf{对比基线}\ — 同时训练 ResNet-34（随机初始化）与
        自定义 SimpleCNN（4 层卷积），定位迁移学习收益与网络容量收益。
  \item \textbf{可解释性}\ — 基于 Grad-CAM\cite{selvaraju2017gradcam} 生成
        ResNet 最后一个残差块的类激活热力图。
\end{itemize}

\section{图像分类模型}

\subsection{ResNet-34 主模型}

\textbf{ResNet-34}\cite{he2016resnet} 由 16 个残差块（共 34 层）构成，
引入\textbf{残差连接}（skip connection）缓解深层网络的梯度消失问题。
相比 ResNet-50 等更深版本，其参数量仅 21.3M，在速度与精度间取得良好平衡，
非常适合作为教学项目主干。

网络结构主要包含：

\begin{itemize}[leftmargin=*]
  \item \texttt{conv1}\ — $7 \times 7$ 大感受野卷积，stride=2；
  \item \texttt{layer1--layer4}\ — 四组残差块堆叠，分别包含 3/4/6/3 个残差单元；
  \item \texttt{avgpool}\ — 全局平均池化；
  \item \texttt{fc}\ — 全连接层，输出 $N$ 维 logits。
\end{itemize}

\textbf{迁移学习适配}\ 为适配 37 类分类任务，保留骨干权重，仅替换最后的
全连接层 \texttt{fc}：将原 ImageNet 的 1000 分类层替换为
\texttt{nn.Linear(512, 37)}。所有层参数均参与训练，但迁移学习提供了高质量的
初始化特征表示。

\subsection{SimpleCNN 自定义基线}

为与 ResNet-34 架构形成容量对比，本项目实现了一个自定义的
\textbf{SimpleCNN}（共 4 层卷积 + 2 层全连接），结构如下：

\begin{center}
\begin{tabular}{llc}
  \toprule
  \textbf{层} & \textbf{参数} & \textbf{输出尺寸} \\
  \midrule
  Conv2d + BN + ReLU + MaxPool & kernel=3, ch=32  & $112 \times 112 \times 32$ \\
  Conv2d + BN + ReLU + MaxPool & kernel=3, ch=64  & $56 \times 56 \times 64$ \\
  Conv2d + BN + ReLU + MaxPool & kernel=3, ch=128 & $28 \times 28 \times 128$ \\
  Conv2d + BN + ReLU + MaxPool & kernel=3, ch=256 & $14 \times 14 \times 256$ \\
  GlobalAvgPool + Flatten      & ---              & $256$ \\
  Linear + ReLU + Dropout(0.5) & 256 $\to$ 128    & $128$ \\
  Linear                       & 128 $\to$ 37     & $37$ \\
  \bottomrule
\end{tabular}
\end{center}

SimpleCNN 参数量约 \textbf{0.37M}，仅为 ResNet-34 的 1/50。它采用纯随机初始化
（He\cite{he2015kaiming} 初始化），不依赖任何预训练权重，构成一个
\textbf{无迁移 + 小容量}的强对比基线。

\subsection{损失函数与优化器}

三个分类模型均采用\textbf{交叉熵损失}（Cross Entropy Loss），公式如下：

\begin{equation}
  \mathcal{L}_{\text{cls}} = -\frac{1}{N}\sum_{i=1}^{N}\sum_{c=1}^{C}
    y_{i,c} \log \hat{y}_{i,c}
\end{equation}

其中 $N$ 为 batch 大小，$C = 37$ 为类别数，$y_{i,c}$ 为 one-hot 真实标签，
$\hat{y}_{i,c}$ 为 softmax 后的预测概率。

优化器采用 \textbf{Adam}\cite{kingma2014adam}，学习率 $\eta = 10^{-3}$，
$\beta_1 = 0.9$，$\beta_2 = 0.999$。使用 \texttt{ReduceLROnPlateau} 调度策略：
验证集指标 $k$ 轮不提升则学习率乘 $0.5$，配合早停避免过拟合。

\section{图像分割模型}

\subsection{U-Net 架构}

\textbf{U-Net}\cite{ronneberger2015unet} 是一种经典的编码器-解码器架构，
其特点在于\textbf{跳跃连接}（skip connection）：将编码器各层的高分辨率特征
与解码器对应层拼接，兼顾定位精度与语义信息。

本项目使用 \texttt{segmentation-models-pytorch} 库提供的 U-Net 实现，
编码器替换为 ResNet-34（\texttt{encoder\_name="resnet34"}），
\texttt{encoder\_weights="imagenet"} 表示编码器从 ImageNet 预训练参数初始化。
解码器默认为 5 层上采样块，输出单通道（二值分割）。

\subsection{损失函数}

分割任务采用 \textbf{BCEWithLogitsLoss}（带 logits 的二元交叉熵）：

\begin{equation}
  \mathcal{L}_{\text{seg}} = -\frac{1}{HW}\sum_{h,w}
    \left[
      y_{h,w} \log \sigma(\hat{y}_{h,w}) +
      (1 - y_{h,w}) \log (1 - \sigma(\hat{y}_{h,w}))
    \right]
\end{equation}

其中 $\sigma(\cdot)$ 为 sigmoid 函数，$y_{h,w} \in \{0, 1\}$ 为真实掩膜，
$\hat{y}_{h,w}$ 为网络输出的 logit。使用 \texttt{WithLogits} 版本比先 sigmoid
再 BCE 数值稳定性更好。

\subsection{评估指标}

除 Pixel Accuracy 外，分割任务还采用 \textbf{Dice 系数}与 \textbf{mIoU} 两个更
能反映前景分割质量的指标：

\begin{equation}
  \text{Dice} = \frac{2 \cdot |P \cap G|}{|P| + |G|},\quad
  \text{IoU}  = \frac{|P \cap G|}{|P \cup G|}
\end{equation}

其中 $P$ 为预测前景像素集合，$G$ 为真实前景像素集合。
\textbf{mIoU}（mean IoU）为前景类与背景类 IoU 的平均。

\section{Grad-CAM 可解释性}

\textbf{Grad-CAM}（Gradient-weighted Class Activation Mapping）
\cite{selvaraju2017gradcam} 通过反向传播目标类的梯度到指定卷积层，
计算每个通道的权重，再将权重与激活加权求和得到类激活热力图。本项目对
\textbf{ResNet-34 的最后一个残差块}（\texttt{layer4[-1]}）生成 Grad-CAM。

算法流程：

\begin{enumerate}[leftmargin=*]
  \item 在 \texttt{layer4[-1]} 注册 forward hook 与 backward hook；
  \item 前向计算得到 Top-1 类 $c^*$ 的 logit；
  \item 对 $c^*$ 反向传播，获得目标层梯度 $\partial y_{c^*} / \partial A^k$；
  \item 按空间维度全局平均池化梯度，得到通道权重：
        $\alpha_k^{c^*} = \frac{1}{HW}\sum_{h,w} \frac{\partial y_{c^*}}{\partial A_{h,w}^k}$；
  \item 加权激活并 ReLU 得到热力图：
        $L^{c^*} = \text{ReLU}\left(\sum_k \alpha_k^{c^*} A^k\right)$；
  \item 上采样到原图尺寸，用 JET 调色板渲染，与原图按 $0.55:0.45$ 的 alpha 混合。
\end{enumerate}

该热力图红色区域表示模型判定该类的关键依据区域，蓝色区域为无关区域，
可用于验证模型是否关注正确的判别特征（如脸部、耳朵、体色等）。

\section{评估指标}

\subsection{分类任务指标}

\begin{itemize}[leftmargin=*]
  \item \textbf{Accuracy}\ — 预测正确样本占总样本比例：
        $\text{Acc} = \frac{1}{N}\sum_{i=1}^{N}\mathbb{1}(\hat{y}_i = y_i)$。
  \item \textbf{Precision (macro)}\ — 对每类计算精确率后取算术平均。
  \item \textbf{Recall (macro)}\ — 对每类计算召回率后取算术平均。
  \item \textbf{F1 (macro)}\ — 对每类计算 F1 后取算术平均。
\end{itemize}

三个宏平均指标更能反映模型在\textbf{少数类别}上的表现（类别均衡时，

\subsection{分割任务指标}

\begin{itemize}[leftmargin=*]
  \item \textbf{Dice 系数}\ — 衡量预测与真实前景的重叠度，更关注前景像素。
  \item \textbf{mIoU}\ — 前景类与背景类 IoU 的平均。
  \item \textbf{Pixel Accuracy}\ — 像素级预测正确率（受类别不均衡影响，作为辅助指标）。
\end{itemize}

        精度提升 \textbf{30.3 个百分点}，成为本项目达到 98.65\% 高精度的\textbf{关键}。
  \item \textbf{网络容量有明显影响}\ — SimpleCNN 与 ResNet-34（随机初始化）同样无迁移，
        但容量差异 50 倍使得准确率差距仍达 17 个百分点。
\end{enumerate}

\section{训练曲线}

完整训练曲线可在系统 \texttt{/analysis} 页面交互查看，其关键观察包括：

\begin{itemize}[leftmargin=*]
  \item \textbf{ResNet-34 预训练}\ —
        训练 Loss 从 0.67 平稳下降至 0.06，验证 Accuracy 第 5 轮即达 0.849，
        至第 100 轮收敛于 0.926；由于测试集样本独立于验证集且置信度较高的样本占比更大，
        最终测试集 Accuracy 达到 \textbf{0.9865}。未观察到明显过拟合。
  \item \textbf{ResNet-34 随机初始化}\ —
        无预训练权重时收敛显著缓慢，最终测试集 Accuracy 仅 0.6831。
        这一现象说明\textbf{无预训练的深层网络难以在小数据集上充分训练}。
  \item \textbf{SimpleCNN}\ —
        前 5 轮验证 Accuracy 从 0.08 快速提升至 0.16，随后每 10 轮约提升 0.10，
        第 50 轮收敛于 0.547，测试集上达到 0.5115。容量瓶颈使其无法挖掘更细粒度特征。
  \item \textbf{U-Net}\ —
        Dice 从第 1 轮的 0.853 稳步提升至第 25 轮的 0.914，此后进入平台期；
        测试集 Dice 达到 \textbf{0.9374}。
\end{itemize}

\subsection{推理服务流水线}

以 \texttt{/api/infer/multitask} 为例，单次调用执行以下步骤：

\begin{enumerate}[leftmargin=*]
  \item \textbf{文件校验}\ — 检查 \texttt{Content-Type} 以 \texttt{image/} 开头，
        否则返回 400；
  \item \textbf{解码}\ — \texttt{utils/image\_io.py} 读取字节流为 PIL Image；
  \item \textbf{分类推理}\ — \texttt{classify\_service} 预处理 + forward +
        softmax，输出 Top-1 与 Top-5 候选；
  \item \textbf{分割推理}\ — \texttt{segment\_service} 预处理 + forward +
        sigmoid + 二值化，输出 mask；
  \item \textbf{可视化渲染}\ — \texttt{utils/visualize.py} 生成叠加图，
        写入 \texttt{tmp/} 并返回静态 URL；
  \item \textbf{Grad-CAM 异步生成}\ — 前端在主结果返回后异步请求
        \texttt{/api/infer/gradcam}。
\end{enumerate}

\subsection{推理页三模式}

\texttt{/predict} 是系统核心交互页面，提供三种互斥的推理模式，

\end{itemize}

\section{混淆矩阵分析}

对 ResNet-34 主模型在 37 类测试集上的 $37 \times 37$ 混淆矩阵进行分析，
结果可在系统 \texttt{/analysis} 页面以热力图交互查看。
关键观察如下：

\begin{itemize}[leftmargin=*]
  \item 大部分品种对角线值接近 20（= 测试集每类样本数），即准确率 100\%；
  \item 仅出现少量 1 张级别的零星混淆对，集中在同一大类内部的亲缘品种之间，
        如\textbf{American Bulldog $\leftrightarrow$ American Pit Bull Terrier}、
        \textbf{Boxer $\leftrightarrow$ Staffordshire Bull Terrier} 等体型相近的犬种对，
        以及 \textbf{Birman $\rightarrow$ Ragdoll}（均为长毛大型猫）；
  \item 犬科之间的混淆多发生于同一体型段（大型猎犬、小型梗犬）；
  \item 仅 1 类（German Shorthaired）Accuracy $= 90\%$，其余 36 类均 $\geq 95\%$，
        表明模型在各品种上的表现相对均衡。
\end{itemize}

  \centering

  \caption{ResNet-34 主模型在测试集上的 $37 \times 37$ 混淆矩阵}

图 \ref{fig:confusion-matrix} 的混淆矩阵热力图中，对角线呈现明显的高亮深色带，即模型在 37 类上的绝大多数品种均达到

\section{错误案例分析}

系统 \texttt{/analysis} 页面展示了 5 张正确案例与 5 张典型错误案例。
通过人工观察错误案例，可归纳出以下常见失败模式：

\begin{enumerate}[leftmargin=*]
  \item \textbf{姿态遮挡}\ — 宠物身体被大范围遮挡（如藏在箱内），
        模型只能看到有限局部特征；
  \item \textbf{视角不寻常}\ — 俯视、仰视或极近距离特写偏离训练分布；
  \item \textbf{亲缘品种}\ — 前述高错品种对的典型误判；
  \item \textbf{光照极端}\ — 过曝或强逆光导致纹理丢失。
\end{enumerate}

错误反馈闭环模块（\texttt{/api/feedback}）让用户可以标注误判案例，
为后续数据扩充与针对性再训练提供素材。

\section{Grad-CAM 可解释性分析}

对若干典型样本生成 Grad-CAM 热力图并观察：

\begin{itemize}[leftmargin=*]
  \item \textbf{短毛猫品种}\ — 热力图普遍集中在\textbf{面部（眼鼻 + 耳）}区域，
        符合人类识别的直觉；
  \item \textbf{长毛狗品种}\ — 热力图分布更分散，覆盖面部与躯干毛色纹理区域；
  \item \textbf{错误案例}\ — 部分错误样本的热力图偏向背景（如草地、家具），
        表明模型被背景干扰，可作为数据增强策略优化的依据。
\end{itemize}

该分析表明模型的判别依据整体合理，符合生物学特征，少数错例暴露出
背景干扰这一潜在改进方向。

\section{模型局限性}

尽管主模型取得 98.65\% 的高准确率，但仍存在若干局限：

\begin{enumerate}[leftmargin=*]
  \item \textbf{少量亲缘品种}\ — 如 American Bulldog 与 American Pit Bull Terrier 等同体型犬种对之间的混淆难以通过现有数据消除；
  \item \textbf{小目标/遮挡场景}\ — 宠物占比过小或被大面积遮挡时分类置信度显著下降；
  \item \textbf{分布外样本}\ — 37 类之外的品种或非宠物图片仍会被强制归类为某一已有类；
  \item \textbf{分割边界}\ — U-Net 在长毛品种的毛发边缘处仍有锯齿状误差；
  \item \textbf{推理速度}\ — CPU 单张推理约 200--300 ms，高并发时需 GPU 加速；
  \item \textbf{可解释性深度}\ — Grad-CAM 仅反映单层激活，难以解释全链路决策。
\end{enumerate}

\section{改进方向}

基于上述局限，后续可从以下方向改进：

\begin{itemize}[leftmargin=*]
  \item \textbf{架构升级}\ — 更换为 ResNet-50 / ConvNeXt / ViT 等更强主干；
  \item \textbf{数据扩充}\ — 针对高错品种主动收集更多样本与更多姿态；
  \item \textbf{损失函数}\ — 分割任务引入 Dice Loss + BCE 联合损失，提升边界质量；
  \item \textbf{多任务联合训练}\ — 共享主干 + 双头输出，实现真正意义上的单次前向；
  \item \textbf{模型蒸馏}\ — 把 ResNet-34 蒸馏到更轻的 MobileNet 便于边缘部署；
  \item \textbf{自监督预训练}\ — 在未标注宠物图片上做对比学习，进一步提升表征质量。
\end{itemize}

  \item \textbf{Grad-CAM 异步集成}\ — 主结果即时返回，热力图后台异步生成，
        用户体验无阻塞；
  \item \textbf{完整工程化落地}\ — 不止于 notebook 跑通，而是提供了
        容器化部署、自动化测试、构建优化、安全加固等一套生产级实践。
\end{itemize}

\section{展望}

若在本项目基础上继续深入，建议从以下方向展开：

\begin{enumerate}[leftmargin=*]
  \item \textbf{真正的多任务联合训练}\ — 使用共享主干 + 双头（分类 head + 分割 head）
        端到端同时训练，进一步降低推理成本并研究任务互补性；
  \item \textbf{更强的可解释性}\ — 引入 Integrated Gradients / LIME 等方法
        横向对比，或引入 attention map 可视化；
  \item \textbf{大规模无标注数据}\ — 使用 MoCo / SimCLR 等自监督方法在
        iNaturalist 等更大规模的生物数据集上预训练；
  \item \textbf{部署优化}\ — 模型蒸馏 + ONNX / TensorRT 导出，
        目标在边缘设备（树莓派、手机）上达到实时推理；
  \item \textbf{错误反馈驱动迭代}\ — 将 \texttt{/api/feedback} 收集到的数据
        周期性回流训练集，形成数据飞轮。
\end{enumerate}
